# i18n

Mika ships with full internationalization at two levels:

1. **Engine-level translations** — built-in UI strings (table labels, buttons, form validation messages, etc.)
2. **App-level overrides** — your own content strings, entity labels, and custom UI text

Both use `@ngx-translate/core` under the hood.

## Setup

Add `i18n` to your `MikaAppConfig`:

```typescript
i18n: {
  defaultLang: 'en',
  fallbackLang: 'en',
  i18nPath: '/assets/i18n/',  // trailing slash required
}
```

Create translation files at `apps/my-app/public/assets/i18n/`:

```
/assets/i18n/
  en.json
  ar.json
  fr.json
```

## Translation file format

```json
// en.json
{
  "COMMON": {
    "SAVE": "Save",
    "CANCEL": "Cancel",
    "DELETE": "Delete"
  },
  "POSTS": {
    "TITLE": "Posts",
    "CREATE": "New Post"
  }
}
```

The engine's built-in keys are loaded via `HybridLoader` and merged with your app's
translation file. Your keys take precedence over engine defaults.

## Changing the active language

```typescript
import { MikaI18nService } from '@mikaverse/core';

const i18n = inject(MikaI18nService);
i18n.setLanguage('ar');
```

This updates the active language, triggers RTL layout (for Arabic, Hebrew, etc.),
and re-renders all translated strings.

## RTL support

When you switch to an RTL language (Arabic, Hebrew, Farsi), Mika automatically:

- Sets `dir="rtl"` on the document
- Mirrors the layout (sidebar, toolbar, form layout)
- Adjusts Material component direction via `Directionality`

No extra config needed.

## Localizable fields

Mark individual entity fields as `localizable: true` to store and display per-language values:

```typescript
{ key: 'title', type: 'text', localizable: true }
```

In the form, localizable fields render per active language. Mika stores them as:

```json
{ "en": "Hello World", "ar": "مرحبا بالعالم" }
```

And reads the correct value for the current locale at render time.

## Engine translation keys

Engine built-in keys are loaded from `@mikaverse/ui-material/assets/i18n/`. You can
override any engine key in your app's translation file:

```json
// en.json — override the default "Save" button label for this app
{
  "MIKA": {
    "FORM": {
      "SAVE": "Publish"
    }
  }
}
```

## Supported languages (demo)

The demo app ships with `en` and `ar` translations. Add more by creating the appropriate
`assets/i18n/<lang>.json` file and calling `i18n.setLanguage('<lang>')`.
